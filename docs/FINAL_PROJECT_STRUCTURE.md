# HALT-RAG Final Project Structure

`HALT-RAG-FINAL/` is the **clean final runnable system**. The repository
root above it (`D:/SJSU/Semester-3/CMPE258/project/`) preserves legacy
experiments, notebooks, prototype modules, duplicate folders, proposal
PDFs, and screenshots - none of those are required to run the final demo.

## Final Backend Path

- `backend/main.py` - FastAPI app for `uvicorn backend.main:app --reload --port 8000`
- `backend/api.py` - compatibility wrapper used by older tests / notebooks
- `src/models/embedding_model.py` - MiniLM embedding wrapper with local fallback
- `src/models/nli_model.py` - HuggingFace NLI verifier with safe fallback
- `src/retrieval.py` - dense FAISS retrieval and index persistence
- `src/generation.py` - Gemini generation with extractive fallback
- `src/deep_hallucination_detector.py` - sentence-level grounding and localization
- `src/hallucination_classifier.py` - answer-level hallucination type and risk
- `src/detector_validation.py` - multi-signal detector confidence
- `src/confidence.py` - final trust score calculation
- `src/knowledge_update.py` - dynamic document and Q&A updates
- `src/audit.py` - JSONL audit logging
- `src/evaluate_detector.py` - curated detector evaluation
- `src/ablation.py` - detector signal ablation study
- `src/utils.py` - shared utilities
- `src/config.py` - runtime configuration

## Final Frontend Path

- `frontend/` - Next.js 14 + Tailwind CSS dark-theme dashboard
  - `app/page.tsx` - Dashboard
  - `app/ask/page.tsx` - Ask console (answer, sources, sentence grounding, detector)
  - `app/admin/page.tsx` - Knowledge update (document or Q&A)
  - `app/logs/page.tsx` - Audit log table with filters
  - `app/about/page.tsx` - Methodology and the professor-question answer
  - `components/` - reusable UI (sidebar, topbar, badges, confidence bar, sentence/source panels, detector panel)
  - `lib/api.ts` - typed API client with `queryHALT`, `addDocument`, `addQA`, `getLogs`, `healthCheck`

## Final Data and Runtime Paths

- `data/corpus/` - final corpus directory and FAISS index files
- `data/sample_corpus.json` - fallback corpus when `data/corpus/` has no source docs
- `data/eval/halt_rag_eval.csv` - 40 curated detector examples
- `logs/audit.jsonl` - runtime audit log generated by the backend
- `results/` - evaluation report, confusion matrix, type / confidence distributions, ablation CSV
